## Quick Start

Get up and running in under 2 minutes:

### Step 1: Install Globally
Install the Ironclad Engine CLI:

```bash
npm install -g ironclad-master-framework
```

### Step 2: Initialize in Your Project
Set up the Domain-Driven structure and baseline configs in any repository:

```bash
ironclad init
```

### Step 3: Run the MCP Server
Connect your AI harness (Claude Code, Cursor, Gemini) to the Ironclad kernel via MCP:

```bash
ironclad mcp
```

### Reset / Uninstall Ironclad
To completely remove Ironclad tracking and configs from your local repository:

```bash
ironclad uninstall
```

## Cross-Platform Support

This framework fully supports **macOS, Linux, and Windows**, with native integrations for:

- **Claude Code**: Integrates natively via MCP and `CLAUDE.md`.
- **Cursor**: Uses `.cursorrules` mapped to the Ironclad 5-Step Operational Loop.
- **Gemini CLI**: Seamless tracking via `.gemini/GEMINI.md` and shared instruction sets.
- **Windsurf**: Compatible via `AGENTS.md` and standard system prompt overrides.

### Token Optimization & Compression

AI context limits are expensive. Ironclad's `WatchService.compressContext()` engine provides aggressive AST-equivalent stripping, achieving up to 32% token savings by safely stripping:
- Multi-line block comments and single-line comments.
- AI-irrelevant logging statements (`console.log`, `console.debug`).
- Consecutive blank lines and excess internal whitespace.

## Key Concepts

### The God-Tier Operational Loop (MANDATORY)

Every task MUST progress through the **Universal Synthesis Loop**. Failure to comply results in terminal slop rejection.

1.  **Understand**: Map architectural dependencies via `Understand-Anything`.
2.  **Plan**: Draft strategic SPARC specifications via `StrategicPlanningService`.
3.  **Delegate**: Spawn parallel agent swarms for multi-file implementation.
4.  **Implement**: Apply cinematic frontend standards & Karpathy-simplicity logic.
5.  **Verify**: Autonomous visual QA and Truth Factor validation via `AuditService`.

### 0.95 Truth Factor
Ironclad grades every file. If a file scores below 95/100 on correctness, security, or maintainability, the framework enforces an automatic rollback. AI-generated code is guilty until proven innocent.

## FAQ

<details>
<summary><b>My AI context window is shrinking quickly.</b></summary>
The Ironclad `WatchService` is designed to mitigate this. Ensure your assistant is using `ironclad watch` or actively leveraging the AST-stripping token compression module to drop unnecessary logging and comments before feeding codebase context back into the LLM.
</details>

<details>
<summary><b>Does this work with Cursor / Windsurf / Gemini?</b></summary>
Yes. Ironclad is a framework that acts as the "operating system" beneath the harness. The rules in `AGENTS.md` and `.cursorrules` ensure the AI complies regardless of the IDE interface you use.
</details>

<details>
<summary><b>How does AgentDB work?</b></summary>
AgentDB uses `better-sqlite3` locally to store JSON and vector representations of architectural decisions. When an agent loses track of the project's strategy, it retrieves the `ROADMAP.json` state mapped by the `StrategicPlanningService`.
</details>
