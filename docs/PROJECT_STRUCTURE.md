# 🏗️ PROJECT STRUCTURE
### **The Modular Anatomy of an Elite Operating System**

The **Ironclad Master Framework** is organized into a modular, high-tier intelligence system. This document outlines the purpose of each directory to ensure absolute clarity during development.

---

## 📂 Root: The Command Center
- **`README.md`**: The cinematic entry point and project overview.
- **`SKILL_ROUTER.md`**: The mandatory Skill-to-Task mapping engine.
- **`Makefile`**: Automation, building, and maintenance triggers.
- **`CLAUDE.md` / `GEMINI.md`**: Sovereign Operational Mandates for AI agents.
- **`package.json`**: Global distribution and dependency management.

## 📂 `.ai-core/`: The Intelligence Hub
This is the **Sovereign Brain** of the framework, housing all consolidated intelligence.
- **`skills/`**: Atomic, specialized toolsets (e.g., `ui-ux-pro-max`, `diagnose`).
- **`agents/`**: Persona-based decision models for complex swarms.
- **`rules/`**: The "Anti-Slop" guidelines and governance mandates.
- **`plugins/`**: MCP and external hardware-accelerated tool integrations.

## 📂 `src/`: The Core Engine
- **`cli/`**: The Command Line Interface implementation.
- **`mcp/`**: The Model Context Protocol server for AI assistants.
- **`core/`**: The DDD-aligned business logic (Domains, Application, Kernel).
- **`scanners/`**: Autonomous audit engines (Security, Architecture, Performance).
- **`scoring/`**: The Truth Score calculation and verification engine.

## 📂 `plans/`: The Strategic Store
- Stores immutable SPARC Specifications generated by the `/plan` command. This is the **Persistent Memory** of architectural intent.

## 📂 `docs/`: Strategic Intelligence
- **`MANIFESTO.md`**: The core philosophy of Ironclad engineering.
- **`SYSTEM_DESIGN.md`**: Deep dive into the 4-layer governance model.
- **`UNIVERSAL_ORCHESTRATION.md`**: The God-Tier Operational Loop.
