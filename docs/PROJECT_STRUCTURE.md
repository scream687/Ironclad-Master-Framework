# 🏗️ Ironclad Architecture & Project Structure

The **Ironclad Master Framework** is organized into a modular, high-tier intelligence system. This document outlines the purpose of each directory to ensure absolute clarity during exploration.

## 📁 Root Directory
- **`README.md`**: The cinematic entry point and project overview.
- **`GEMINI.md`**: The Ironclad Operational Mandates (AI source of truth).
- **`SKILL_ROUTER.md`**: The mandatory Skill-to-Task mapping engine.
- **`Makefile`**: Automation and maintenance commands.
- **`CONTRIBUTING.md`**: Guidelines for elite contributors.
- **`LICENSE`**: Business Source License 1.1.

## 📁 `.ai-core/` (The Intelligence Hub)
This is the "brain" of the framework, containing all scraped assets from the system.
- **`skills/`**: Atomic, specialized toolsets (e.g., `ui-ux-pro-max`, `diagnose`).
- **`agents/`**: Persona-based decision models.
- **`rules/`**: Linting, formatting, and anti-slop guidelines.
- **`plugins/`**: MCP and external tool integrations.
- **`commands/`**: Custom shell scripts and automation triggers.

## 📁 `docs/` (Strategic Documentation)
- **`MANIFESTO.md`**: The philosophy behind the Ironclad Framework.
- **`WORKFLOW.md`**: Deep dive into the SPARC execution loop.

## 📁 `assets/`
- Visual branding, banners, and diagrams for the repository.
