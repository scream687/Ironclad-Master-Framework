# 🏗️ SYSTEM ARCHITECTURE
### **The God-Tier Blueprint for Autonomous Precision**

The **Ironclad Master Framework** is a recursive, multi-layered intelligence system that separates **Intent** from **Implementation**. It is designed to transform a raw LLM into a disciplined Senior Engineer.

---

## 🍱 The 4-Layer Governance Model

### 1. The Sovereign Layer (The Rules)
- **Files**: `GEMINI.md`, `CLAUDE.md`, `.aiderules`, `.cursorrules`.
- **Function**: These are the fundamental mandates. They force the AI agent into a state of **High Compliance** the moment the environment is initialized.

### 2. The Strategic Router (The Brain)
- **File**: `SKILL_ROUTER.md`.
- **Function**: An intelligent task-to-tool mapping engine. It intercepts user intent and routes it to the most specialized `.ai-core` intelligence asset.

### 3. The Intelligence Hub (The Skills)
- **Location**: `.ai-core/`.
- **Function**: The persistent memory and capability store. It houses "Atomic Skills" (e.g., `tdd`, `design-taste`) and specialized Persona Agents that execute the work.

### 4. The Enforcement Layer (The Audit)
- **Tools**: `ironclad audit`, `Makefile`, `.husky/`.
- **Function**: The zero-tolerance validation layer. It rejects any output that fails the **0.95 Truth Factor** or violates a governance mandate.
