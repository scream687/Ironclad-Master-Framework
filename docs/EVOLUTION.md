# 🔄 Ironclad Evolution Engine

The **Ironclad Evolution Engine** ensures that the framework never becomes stagnant. It is designed to adapt to any project context and upgrade its own intelligence based on real-world performance.

## 1. MVP-to-Enterprise Adaptability
The framework operates in three "Adaptability Modes" depending on the project's maturity:

| Mode | Trigger | Focus | Mandatory Skills |
|---|---|---|---|
| **⚡ Blitz (MVP)** | New Repo / Prototype | Speed + Core UX | `prototype`, `sparc-methodology` |
| **🏗️ Scale (Growth)** | Feature Expansion | Architecture + Stability | `improve-codebase-architecture`, `tdd` |
| **🛡️ Ironclad (Enterprise)** | Production / Security | Hardening + Compliance | `V3 Security Overhaul`, `Verification & QA` |

## 2. Self-Upgrade Loop
The framework upgrades itself through a 3-step recursive loop:
1. **Analyze:** After every major task, run `make audit` to detect patterns of "slack" or inefficiency.
2. **Distill:** Identify if a new skill or rule is needed. Use the `Skill Builder` to generate it.
3. **Commit:** Merge the new intelligence into `.ai-core/`.

## 3. Remote Intelligence Fetching
The framework can autonomously fetch specialized skills from the global community.
- **Trigger:** When a user requests a skill not found in `.ai-core/`, or when a task requires extreme specialization (e.g., "Web3 Security").
- **Action:** Execute `make fetch-skill REPO=user/repo`.
- **Validation:** After fetching, the AI MUST run `make audit` to ensure the new skill adheres to Ironclad anti-slop standards.

## 4. Contextual Adaptation
Before starting any new project, the AI MUST perform a **Context Audit**:
- **Tech Stack Check:** Detects framework (Next.js, Vite, etc.) and adjusts rules.
- **Complexity Check:** Adjusts the SPARC depth (Blitz for small tasks, Deep for architectural shifts).
- **Security Tier Check:** Adjusts the level of verification required.
