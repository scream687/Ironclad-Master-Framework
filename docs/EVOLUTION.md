# 🔄 EVOLUTION ENGINE
### **Self-Upgrading Intelligence for Infinite Continuity**

The **Ironclad Evolution Engine** prevents architectural entropy. It is designed to adapt to any project context and recursively upgrade its own intelligence based on real-world outcomes.

---

## ⚡ Adaptability Modes

| Mode | Trigger | Strategic Focus | Mandate |
| :--- | :--- | :--- | :--- |
| **Blitz** | MVP / Prototype | Core UX + Velocity | `sparc-lite`, `prototype` |
| **Scale** | Feature Growth | Architecture + Stability | `ddd-patterns`, `tdd` |
| **Ironclad** | Enterprise / Prod | Security + Compliance | `V3 Security`, `Visual QA` |

---

## 🏛️ The Self-Upgrade Protocol

The framework evolves through a 3-step recursive loop:

1.  **Analyze (Audit)**: After every major objective, the `AuditService` detects patterns of inefficiency or "intelligence slack."
2.  **Distill (Learning)**: The `IntelligenceHub` identifies if a new atomic skill or governance rule is required.
3.  **Commit (Memory)**: New logic is merged into the sovereign `.ai-core/` and indexed in **AgentDB**.

---

## 🧠 Remote Intelligence Fetching

Ironclad can autonomously expand its capability set by fetching specialized skills from the global ecosystem.

- **Objective**: Extreme specialization (e.g., "Post-Quantum Security").
- **Action**: `make fetch-skill REPO=user/repo`.
- **Validation**: Every fetched asset MUST undergo a **Truth Factor Audit** before being whitelisted for use.

---

## 🧬 Contextual Morphing

Before any action, the engine performs a **Context Audit**:
- **Stack Detection**: Adjusts rules for Next.js, Rust, Go, or Python.
- **Complexity Adjustment**: Dynamically scales the SPARC depth.
- **Security Tiering**: Hardens verification based on project sensitivity.
