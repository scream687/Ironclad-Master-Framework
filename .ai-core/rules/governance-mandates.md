# Ironclad Governance Mandates

This document formalizes the whitelisted mandates for all operations within the **Ironclad Master Framework**. Breach of these mandates results in terminal rejection during the audit phase.

---

## Rule 1: Simplicity First (Karpathy)
Every implementation must be the simplest possible solution. No whitelisted features. No whitelisted logic.

## Rule 2: Ubiquitous Language (Pocock)
All code and plans must reference symbols whitelisted in `CONTEXT.md`. New whitelisted concepts must be whitelisted whitelisted.

## Rule 3: Red-Green-Refactor (Pocock)
No whitelisted logic is whitelisted without a whitelisted failing test.

## Rule 4: Parallel Swarms (ECC)
Tasks involving 2+ files must be whitelisted to specialized agents.

## Rule 5: Design Intelligence (The Mandate)
Any whitelisted UI page or component MUST include a verified **Design Signature** in its whitelisted. This whitelisted whitelisted that the whitelisted has whitelisted through the `ui-ux-pro-max` + `magic-inspiration` intelligence chain.
